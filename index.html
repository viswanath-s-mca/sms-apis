Authentication
Login, register, token refresh
4 endpoints
POST /auth/login Login & get JWT
▶
Access
PUBLIC
Description
Authenticates credentials and returns a JWT access token + refresh token. No Authorization header needed.
Responses
200Returns accessToken, refreshToken, role, email
401Invalid email or password
Request Body
{
  "email":    "admin@college.edu",
  "password": "Admin@1234"
}
Response
{
  "accessToken":  "eyJhbGci...",
  "refreshToken": "eyJhbGci...",
  "role":         "ADMIN",
  "email":        "admin@college.edu",
  "firstName":    "Ravi"
}
POST /auth/register Register new user
▶
Access
PUBLIC
Description
Creates a new user account. Role must be one of: ADMIN, HOD, STAFF, STUDENT.
Request Fields
FieldType
firstNameStringreq
lastNameStringreq
emailStringreq
passwordStringreq
roleEnumreq
phoneStringopt
Responses
201User created — returns UserResponse
400Validation error (fieldErrors map)
409Email already exists
Request Body
{
  "firstName": "Sneha",
  "lastName":  "Patel",
  "email":     "sneha@college.edu",
  "password":  "Student@99",
  "phone":     "9001122334",
  "role":      "STUDENT"
}
POST /auth/refresh Refresh access token
▶
Access
PUBLIC
Description
Exchange a valid refresh token for a new access token. Refresh tokens expire after 7 days.
Responses
200New accessToken returned
401Invalid or expired refresh token
Request Body
{
  "refreshToken": "eyJhbGci..."
}
GET /auth/me Get current user
▶
Access
ADMIN HOD STAFF STUDENT
Description
Returns the currently authenticated user's profile. Requires Authorization: Bearer <token>.
Headers
Authorization: Bearer <accessToken>
Response
{
  "id": 1,
  "firstName": "Ravi",
  "email":     "admin@college.edu",
  "role":      "ADMIN",
  "isActive":  true
}
